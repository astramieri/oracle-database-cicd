# 01- Project Initialize

## 1. Initialize the SQLcl Project

```
SQL> project init -name <name> -schemas <schema1>,<schema2> -directory <directory>
```

Initializes a new SQLcl project in the specified directory. The flags are:

- `-name` — the project name.
- `-schemas` — a comma-separated list of database schemas to include in the project.
- `-directory` — the local directory where the project files will be created.

## 2. Initialize a Git Repository

```
SQL> !git init --initial-branch=main
SQL> !git add .
SQL> !git commit -m "chore: initializing <name> git repository"
```

- `git init --initial-branch=main` — initializes a new Git repository in the current project directory. The `--initial-branch=main` flag sets the default branch name to `main` instead of the legacy `master`.
- `git add .` — stages all project files generated by `project init` for the first commit.
- `git commit -m` — creates the initial commit with a descriptive message.

## 3. Add a Remote Repository

```
SQL> !git remote add origin https://github.com/<username>/<repository>.git
```

Links the local Git repository to an existing remote repository on GitHub. The remote repository must already exist and be **empty** (no commits, no README, no .gitignore) before running this command. `origin` is the conventional alias for the primary remote.

To verify the remote was added correctly:

```
SQL> !git remote -v
```

Lists all configured remotes with their fetch and push URLs.

## 4. Push to the Remote Repository

```
SQL> !git push -u origin main
```

Pushes the local `main` branch to the `origin` remote. The `-u` flag (short for `--set-upstream`) is required on the **first push** — it establishes a tracking relationship between the local and remote branch, so subsequent pushes can be done with just `git push`.
